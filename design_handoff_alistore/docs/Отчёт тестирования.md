# AliStore — Отчёт о тестировании прототипа

Дата: июль 2026. Метод: загрузка каждого экрана + прогон интеракций в превью.

## Флагманские приложения — ✅ работают
- **Клиент App 2.0** — вход (гость/OTP), главная, каталог, карточка, корзина (добавление, кол-во), оформление (способ→адрес→оплата→подтверждение). Полный путь заказа проходит. Консольных ошибок нет.
- **POS 2.0** — тап товара → чек, оплата с методами КР (нал/карта/MBank/O!Деньги/Bakai/О!Банк/рассрочка M+/Zero), смешанная оплата. Работает. Ошибок нет.
- **Сотрудник App 2.0** — загрузка чистая, смена/заказы/задачи/скупка на месте.
- **ERP 2.0** — дашборд с графиком, навигация по модулям (Склад открывает таблицу остатков со статусами). Работает. Ошибок нет.

## Операционные экраны — ✅ работают
- **Order State Machine** — 12+ статусов кликабельны, детали (роли, условия, кнопки, уведомления, событие Ledger) переключаются.
- **Списание товара** — цикл start→approval→event→final; **guard корректно блокирует** отправку без полного комплекта доказательств (проверено — это правильное поведение, не баг); audit log, уведомления, связанные экраны на месте.

## Консольные ошибки
Не обнаружено ни на одном проверенном экране.

## Замечания
- Все проверенные интеракции срабатывают, состояния переключаются, guard-условия (evidence/approval/лимиты) работают как задумано.
- Прототипы — дизайн-референс: серверные проверки прав/лимитов/событий реализуются в коде (см. handoff-пакет).

## Вывод
Прототип функционально целостен и готов к передаче на разработку. Кликабельная навигация, рабочие сценарии и операционные guard-механики подтверждены.
