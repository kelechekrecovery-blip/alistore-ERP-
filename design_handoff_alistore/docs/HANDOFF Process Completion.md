# AliStore — HANDOFF: Process Completion Pass

Слой, закрывающий бизнес-процессы до уровня операционной системы. Все экраны связаны в хабе «AliStore Обзор проекта».

## Создано в этом проходе

### Карта
- **Process Map 2.0** — 16 процессов: владелец, старт→результат, статусы, экраны, доказательства.

### P0 — ядро операций (12)
- Order State Machine — 12+ статусов заказа: кто переводит, условия, кнопки, уведомления, событие Ledger, видимость по ролям.
- Payment Ledger — единая книга денег: способ, кто принял, где деньги, сверка, привязки к заказу/смене/сотруднику.
- Cash Shift Opening — открытие смены: остаток, методы, чек-лист, ошибки (смена не закрыта / нет прав).
- Cash Shift Closing — закрытие: ожид./факт, расхождение → инцидент, фото, Z-отчёт.
- Approval Rules Matrix — 10 опасных действий: создатель, аппрувер, лимит, риск, доказательство, audit.
- Role Permission Matrix — 9 ролей × права, скрытые данные, 2FA, обучение для допуска.
- Inventory Count — инвентаризация: скан, расхождение, списание, расследование.
- IMEI Lifecycle — timeline устройства приход→…→списание, риск-флаги.
- Courier Cash Handover — сдача COD-денег курьером, сверка по заказам, расхождение.
- Failed Delivery — 7 причин, рекомендация, следующий шаг, фото обязательно.
- Refund Money Flow — статусы requested→reconciled, approval, возврат тем же способом.
- Warranty Case Detail — created→…→closed, SLA, диагностика, ремонт/замена/отказ.
- (Exchange Flow — уже был: «Обмен товара».)

### P1 — CRM/поддержка/поставщики/франшиза (6)
- Support Inbox — все каналы, приоритет, SLA, быстрые ответы, эскалация.
- Notification Preferences — каналы × типы, consent, отписка, частота.
- Marketing Segment Builder — условия → аудитория → прогноз выручки → consent-фильтр.
- Campaign ROI — бюджет→выручка→маржа→ROI, отписки/жалобы, авто-стоп.
- Supplier Scorecard — поставки, задержки, брак, RMA, маржа, риск-скор.
- Franchise Audit Checklist — стандарты, рейтинг, нарушения, план исправления.

### P2 — для разработки (2 экрана + документы)
- API & Data Contracts — 18 сущностей: поля, связи, endpoints, события, idempotency, PII, доступ.
- QA Test Scenarios — 15 приёмочных сценариев по приоритету и риску.
- HANDOFF (этот файл) + Self-Review.

## Обновлено
- Обзор проекта — новые разделы: Process Completion Pass (P0), P1, P2.
- Хаб связывает все новые экраны; главные карточки продуктов ведут на флагманы 2.0.

## Закрытые риски
- Деньги: где каждый сом (Payment Ledger), сверка кассы и курьеров, возврат только через approval.
- Товар: IMEI не продаётся дважды, инвентаризация с расхождениями, списание с approval.
- Доступ: матрица ролей + 2FA + PII-маскирование, опасные действия по матрице.
- Доверие: доказательства (Evidence Vault) на каждом критичном шаге, аудит неизменяем.

## Реализация в коде — порядок
1. AuditEvent (append-only) + Order + Payment + DeviceUnit + Reservation.
2. State-machine заказа + Approval + Role permissions (серверная проверка).
3. CashShift + Courier handover + Refund/Exchange.
4. Inventory + Warranty + Supplier RMA.
5. Support Inbox + Notifications + Campaign/Segment (consent).
6. API по контрактам + QA-сценарии как приёмочные тесты.

Фискализация ГНС — вне системы (ведёт владелец).
