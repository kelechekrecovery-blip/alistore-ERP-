## Ключевые архитектурные решения
- **Event Ledger** — единый источник правды (append-only); все модули пишут события, отчёты/Risk/Command Center читают оттуда → цифры не расходятся.
- **Исправления — компенсирующими событиями**, не правкой истории.
- **Опасные действия** (refund, скидка ниже лимита, списание, компенсация) требуют подтверждения старшего/владельца + причину, всё в аудите.
- **Роли и доступы** назначает владелец; полный лог по каждому пользователю.
- **Graceful degradation** оборудования и сети (Offline POS, ручной fallback).
- **Фискализация ГНС — вне системы** (ведёт владелец), по явному требованию.

## Границы (честно)
- Это **дизайн-прототипы + ТЗ**, не задеплоенный бэкенд. Prisma-модели, транзакции, серверные проверки прав, реальные webhook и тесты реализуются в кодовой базе по handoff-докам.
- Реальные фото товаров ждут материалов владельца (сейчас плейсхолдеры).
- Обучение/Франшиза — на уровне дашборда + один детальный подэкран.

## Порядок реализации в коде
1. Ядро: Order + Reservation + DeviceUnit(IMEI) + Payment + AuditEvent + Event Ledger → закрепить Business Invariants.
2. ReturnCase + Exchange + CashShift (деньги/склад/аудит).
3. Courier + Delivery, Debts/рассрочка, Supplier RMA.
4. Risk Center + Command Center + отчёты (читатели Ledger).
5. Notifications Outbox + Telegram-бот (env-ключи на сервере).
6. AI-слой (оценка Б/У, динамические цены, ассистент) — по API.
7. Hardware-агент (сканер/принтеры/терминал), Импорт данных при запуске.
