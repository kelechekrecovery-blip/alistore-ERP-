# Пакет ревью для юриста — AliStore

Дата сборки: 2026-07-18. Назначение: одна встреча с юристом владельца, на которой
закрываются все вопросы разделов В и Г. Пакет собран из реального кода и документов
репозитория — ничего не выдумано; всё, что требует знания права КР, оставлено пустым
полем или сформулировано как вопрос.

Контекст бизнеса: розница электроники в КР (2 точки + интернет-магазин), новые и б/у
устройства, trade-in/скупка б/у с договором и KYC-фото документов, рассрочка/кредит
через банки-партнёры, партнёрство по ОСАГО, доставка с наложенным платежом (COD).

---

## А. Что система хранит и делает с данными (1 страница, по коду)

Источник истины — PostgreSQL + append-only Event Ledger (`apps/api/src/audit/event-types.ts`).

**Персональные данные покупателей** (`apps/api/prisma/schema.prisma`):

- `Customer.phone` (уникальный, в схеме помечен «PII — маскировать младшим ролям», :471),
  `Customer.name`, флаг маркетингового согласия `Customer.consent` (:473).
- Адреса: `CustomerAddress.title/text/comment` (:492–506); адрес доставки заказа
  `Order.deliveryAddress` (:939).
- Соц-идентичности: `CustomerIdentity.provider/subject/email/displayName/avatarUrl` (:568–578).
- Согласие с офертой и обработкой ПДн на витрине: `Order.piiConsentAt` (:960) — ставится
  только при обязательном чекбоксе на шаге 3 чекаута (по умолчанию не отмечен, без него
  заказ не отправляется; см. `BACKLOG.md` запись `GAP-LEGAL-001`).

**Trade-in / скупка б/у** (`TradeInDevice`, schema.prisma:2526–2542):

- `sellerPassport` — паспорт продавца (защищённое поле + anti-fraud), `contractId`
  (генерируется сервером при создании, `apps/api/src/tradeins/tradeins.service.ts`),
  `imei` (anti-fraud: тот же IMEI в скупке и в продаже → risk-флаг), `model`, `grade`,
  `price`. События ledger: `tradein.assessed`, `tradein.contracted`.

**Evidence Vault (фото документов KYC и прочие вложения)** (`EvidenceUpload`,
schema.prisma:2650–2675):

- Приватные объекты (R2/S3), чтение только по авторизации, каждое чтение аудируется
  (`evidence.accessed`). Флаги `isPii`, `retentionUntil`, поля purge-журнала.
- Автоматическая ретенция: только для trade-in KYC-меток `passport`, `passport_front`,
  `passport_back`, `identity_document`, `tradein_kyc`
  (`apps/api/src/evidence/evidence-retention.policy.ts:10–16`). Срок — 365 дней
  по умолчанию (env `EVIDENCE_PII_RETENTION_DAYS`, допустимо 30–3650), версия политики
  `kg-privacy-v1`. Ежечасный worker удаляет объект, редактирует запись и пишет
  `evidence.purged` (только SHA-256 ссылка); чтение после purge → ошибка `evidence_purged`.
  Политика: `docs/PII-EVIDENCE-RETENTION.md`.
- Прочие evidence (гарантия, сервис, заказы, склад) — **без** авто-ретенции до отдельного
  решения владельца/юриста.

**Удаление аккаунта и экспорт** (`GAP-ACCOUNT-DELETE-001`): самообслуживание
`DELETE /customers/me` (анонимизация ПДн на месте, номер телефона освобождается, заказы/
платежи/ledger сохраняются как append-only история) и `GET /customers/me/export` (JSON).

**Чего система НЕ делает:** не передаёт ПДн третьим лицам из кода (кроме платёжных
провайдеров при оплате и SMS/push-транспортов уведомлений), не имеет клиентской аналитики
(`GAP-ANALYTICS-001`), не генерирует фискальные номера/QR (`GAP-FISCAL-001`).
