# ERP Design Gap Audit

**Цель:** зафиксировать расхождения между дизайн-референсами `design_handoff_alistore/screens/*.dc.html` и текущей реализацией desktop ERP в `apps/web`.

**Scope:** ERP 2.0 core + связанные операционные процессы.

**Источники правды:**
- `design_handoff_alistore/screens/AliStore ERP 2.0.dc.html` — master-референс оболочки и 7 core-модулей
- `design_handoff_alistore/screens/AliStore Складской учёт.dc.html`
- `design_handoff_alistore/screens/AliStore Финансы 2.0.dc.html`
- `design_handoff_alistore/screens/AliStore Аналитика.dc.html`
- `design_handoff_alistore/screens/AliStore HR.dc.html`
- `design_handoff_alistore/screens/AliStore Логистика управление.dc.html`
- `design_handoff_alistore/screens/AliStore Операционка точки.dc.html`
- `design_handoff_alistore/screens/AliStore Сервис-центр.dc.html`
- `design_handoff_alistore/screens/AliStore Маркетинг CMS.dc.html`
- `design_handoff_alistore/screens/AliStore Управление товарами.dc.html`
- `design_handoff_alistore/screens/AliStore Закупки.dc.html`
- `design_handoff_alistore/screens/AliStore Order State Machine.dc.html`
- `design_handoff_alistore/screens/AliStore Безопасность.dc.html`
- `design_handoff_alistore/screens/AliStore Process Map 2.0.dc.html`

**Методология:** read-only сравнение кода и дизайна по 6 слоям: цвета, типографика, отступы/радиусы, layout, интерактив, данные/API.

## Глобальные замечания

1. **Табы:** В HR, Сервис-центре, Операционке, Управлении товарами, Рисках, Финансах, Складе используются underline-табы или их нет, хотя дизайн требует pill-табов (`border-radius: 999px`).
2. **Радиусы:** Внешние карточки часто `rounded-[8px]` вместо `16px`/`18px`.
3. **Токены:** В коде много хардкод-HEX (`#1A1611`, `#2E2822` и т.д.) вместо токенов `surface`, `surface-2`, `surface-3`, `line`, `bright`, `muted`, `subtle`, `faint`. Визуально совпадает, но нарушает единую систему.
4. **Шрифты:** `fonts.ts` и `layout.tsx` настроены корректно (Sora, Golos, JetBrains Mono), но в некоторых местах используются неправильные размеры или шрифты (например, JetBrains Mono для действий в Dashboard).
5. **Данные:** Несколько модулей требуют расширения API или новых полей (KPI/ЗП бонусы/штрафы, сегменты CRM, AI-чат, матрица SKU, cashflow, 2FA и т.д.).
6. **Selection:** `::selection` в `globals.css` использует `#FF5B2E` вместо `#C6FF3D` — глобальное расхождение.
