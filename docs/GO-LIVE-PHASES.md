# Go-Live — фазовый план (прототип → боевой магазин в КР)

> Оркеструет Claude. Софт-ядро готово и зелёное (326+9 E2E). Ниже — «последняя миля»
> до реальных клиентов и денег, разложенная по фазам с **владельцем** и **критерием приёмки**.
> Владельцы: **Claude** (моя lane + оркестрация), **Codex** (auth/payments/receipts/main/infra),
> **User** (аккаунты/договоры/сервер/железо/юр). Порядок — сверху.

Легенда владельца: 🟦 Claude · 🟩 Codex · 🟧 User (внешнее)

---

## G0 — Прод-каркас безопасности (без внешних зависимостей) — СНАЧАЛА
- 🟩 **Fail-fast прод-гард**: приложение НЕ стартует при `NODE_ENV=production`, если `JWT_SECRET`
  дефолтный или `AUTH_OTP_DEV_ECHO=true`. (main.ts / config)
- 🟩 **CORS по allowlist из env** + `helmet` (HSTS/CSP/nosniff/frameguard). Сейчас `enableCors()` нараспашку.
- 🟦 **`.env.production.example`** + чеклист смены всех секретов (JWT, DB, MinIO, Novu…).
- **Приёмка:** `npm run launch:preflight:strict` зелёный на прод-конфиге; старт с дефолт-секретом падает.

## G1 — Боевой платёжный шлюз — 🟧 блокер: договор с провайдером
- 🟩 **PaymentProvider-порт** (createIntent / verifyWebhook / status) + адаптер MBank/O!Деньги/эквайринг:
  реальные API-вызовы, **верификация подписи webhook**, идемпотентность по txnId под гонкой.
  Сейчас `payment-intents.service.ts` — 100% sandbox (`/sandbox/payments/...`, без вызова шлюза).
- 🟧 Аккаунт/ключи провайдера + тест-контур.
- **Приёмка:** реальный платёж боевого провайдера → order `paid`, подпись webhook проверена, двойной webhook не удваивает платёж.

## G2 — Боевой SMS/OTP — 🟧 блокер: SMS-шлюз КР
- 🟩 **SmsSender-порт** + адаптер провайдера (Nikita/SMSC/…); auth OTP шлёт через него;
  в проде echo запрещён. Сейчас OTP только эхом в ответе (`auth.service.ts:57`), реальной доставки нет.
- 🟧 Договор + отправитель (sender name).
- **Приёмка:** реальный клиент получает OTP по SMS и входит; `AUTH_OTP_DEV_ECHO` не действует в проде.

## G3 — Фискализация (ККМ/ОФД КР) — 🟧 блокер: фискальный оператор
- 🟩 **FiscalReceipt-порт** + адаптер оператора; продажа регистрирует фискальный чек (номер+QR налоговой).
  Сейчас чек — обычный PDF, не фискальный документ (фискализации в коде нет).
- 🟧 Договор с ОФД/оператором + сертифицированная касса.
- **Приёмка:** POS-продажа → фискальный чек с QR; сверка с кабинетом налоговой.

## G4 — Устранение находок глубокого аудита — 🟦 Claude + 🟩 Codex (по lane)
- Идёт 4-агентный аудит: **деньги/транзакции · auth/безопасность · данные/масштаб · тесты**.
  Каждая находка CRITICAL/HIGH → фикс + тест; недостающие критичные тесты (двойная продажа IMEI,
  expiry резерва под гонкой, webhook-retry идемпотентность, double-redeem giftcard, IDOR/эскалация).
- **Приёмка:** все CRITICAL/HIGH закрыты; новые тесты зелёные; суита не флейкает в CI.

## G5 — Прод-инфра + чистые данные — 🟧 User (сервер) + 🟦/🟩 (скрипты)
- 🟧 Развернуть `infra/` (docker-compose + Caddy) на сервере: домен, HTTPS, бэкапы (`infra/backup.sh`),
  мониторинг (Sentry/health), автозапуск (systemd/pm2).
- 🟩/🟦 Чистый прод-seed (без демо/QA-мусора) + реальный каталог; проверить backup→restore.
- **Приёмка:** `npm run mvp:verify` зелёный НА СЕРВЕРЕ; восстановление из бэкапа проходит.

## G6 — Внешние активации + запуск — 🟧 User
- AI-ключ OpenRouter → живой LLM. Telegram/WhatsApp токены → каналы. Мобильное в сторы (EAS + Apple/Google).
- **Нагрузочное + пентест.** Юр-документы: **оферта, политика перс.данных, согласие на паспорт в скупке Б/У**,
  фискальные требования, потребзащита. Обучение кассиров. Касса-железо на точке.
- **Приёмка (поэтапно):** пилот (огранич. онлайн) → soft launch (1 точка) → полный запуск.
